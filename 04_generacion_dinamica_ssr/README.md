## üìä Data Fetching - Obtenci√≥n de datos

[Data Fetching](https://nextjs.org/docs/app/getting-started/fetching-data) es el mecanismo que Next.js utiliza para obtener datos del servidor, construido sobre la **API Fetch nativa** que permite obtener datos de forma as√≠ncrona.

### üíæ Sistema de cach√© autom√°tico

**Comportamiento por defecto:** Cada petici√≥n con Fetch se agrega autom√°ticamente a un cach√© inteligente.


**Beneficios del cach√©:**

- ‚ö° **Mejor rendimiento**: Evita peticiones innecesarias
- üåê **Reducci√≥n de latencia**: Respuestas instant√°neas desde cach√©
- üí∞ **Menor costo**: Reduce llamadas a APIs externas
- üîã **Mejor UX**: Carga m√°s r√°pida para el usuario

### üîÑ Revalidaci√≥n de datos

Permite actualizar datos en intervalos espec√≠ficos sin perder los beneficios del cach√©:


**Funcionamiento:**

- üìÖ Los datos se almacenan en cach√© durante el tiempo especificado
- ‚è∞ Al cumplirse el tiempo, Next.js hace una nueva petici√≥n
- üîÑ El cach√© se actualiza con los nuevos datos

### üöÄ Dynamic Data Fetching

Para datos que cambian constantemente, usa `cache: "no-store"`:

```typescript

  cache: "no-store", // üîÑ Siempre datos frescos
});
```

**Casos de uso:**

- üìä **Dashboards en tiempo real**: Datos financieros, anal√≠ticas
- üí¨ **Chats y mensajer√≠a**: Conversaciones actualizadas
- üõí **Inventarios**: Stock de productos en tiempo real

### ‚öôÔ∏è Configuraci√≥n sin Fetch API


## üîç Debugging con Breakpoints

Los **breakpoints** son herramientas profesionales para depuraci√≥n que superan a los `console.log` tradicionales.

### üéØ ¬øQu√© son los breakpoints?

Los breakpoints permiten:

- Escribir: `Debug: Start Debugging`
- Seleccionar: `dev` para modo desarrollo

#### **3. Controles de debugging:**

- ‚è≠Ô∏è **Step Over**: Ejecutar l√≠nea actual
- ‚¨áÔ∏è **Step Into**: Entrar en funciones
- ‚¨ÜÔ∏è **Step Out**: Salir de funci√≥n actual

### üí° Ventajas sobre console.log

- üéØ **Control total**: Pausar exactamente donde necesitas
- üìä **Vista completa**: Variables, scope, call stack
